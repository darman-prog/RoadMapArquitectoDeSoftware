This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics[cite: 1]. Implement real working code with exceptional attention to aesthetic details and creative choices[cite: 1].

The user provides frontend requirements: a component, page, application, or interface to build[cite: 1]. They may include context about the purpose, audience, or technical constraints[cite: 1].

## Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction[cite: 1]:
- **Purpose**: What problem does this interface solve? Who uses it?[cite: 1]
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, Y2K/cyber, neo-brutalist, etc[cite: 1]. Use these for inspiration but design one that is true to the aesthetic direction[cite: 1].
- **Constraints**: Technical requirements (framework, performance, accessibility)[cite: 1].
- **Differentiation**: What makes this UNFORGETTABLE? What's the one thing someone will remember?[cite: 1]

**CRITICAL**: Choose a clear conceptual direction and execute it with precision[cite: 1]. Bold maximalism and refined minimalism both work - the key is intentionality, not intensity[cite: 1].

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Production-grade and functional[cite: 1]
- Visually striking and memorable[cite: 1]
- Cohesive with a clear aesthetic point-of-view[cite: 1]
- Meticulously refined in every detail[cite: 1]

## Frontend Aesthetics Guidelines

Focus on:
- **Typography & Font Injection**: Choose fonts that are beautiful, unique, and interesting[cite: 1]. Avoid generic fonts like Arial and Inter[cite: 1]. **MANDATORY**: You must explicitly import distinctive fonts from Google Fonts via `@import` or `<link>` in your code (e.g., *Syne, Clash Display, Playfair Display, Space Mono, Bricolage Grotesque*). Pair a distinctive display font with a highly legible, characterful body font[cite: 1].
- **Color & Theme**: Commit to a cohesive aesthetic[cite: 1]. Use CSS variables for consistency[cite: 1]. Dominant colors with sharp accents outperform timid, evenly-distributed palettes[cite: 1]. Avoid default #000 or #FFF; use rich, nuanced off-whites and deep, textured darks.
- **Motion & Micro-interactions**: Use animations for effects and micro-interactions[cite: 1]. Prioritize CSS-only solutions for HTML[cite: 1]. Focus on high-impact moments: staggered reveals (animation-delay) on page load[cite: 1]. Every button, card, and input MUST have an elegant, responsive hover, focus, and active state that surprises the user[cite: 1].
- **Spatial Composition**: Unexpected layouts[cite: 1]. Asymmetry[cite: 1]. Overlap[cite: 1]. Diagonal flow[cite: 1]. Grid-breaking elements[cite: 1]. Generous negative space OR controlled density[cite: 1]. Never align everything to a basic center-grid unless the aesthetic explicitly demands strict symmetry.
- **Backgrounds & Visual Details**: Create atmosphere and depth rather than defaulting to solid colors[cite: 1]. Apply creative forms like gradient meshes, CSS noise textures, geometric SVG patterns, layered transparencies, dramatic layered shadows, decorative borders, custom cursors, and grain overlays[cite: 1].

## Technical Quality & Reality Check

To prevent creative UI from breaking functionality, you MUST ensure:
1. **Real UX/UI Usability**: Forms must have visible labels, inputs must be clickable, text must meet WCAG contrast guidelines, and navigation must be logical. Creative does not mean broken.
2. **Component Completeness**: Do not leave `// TODO` or placeholder code. If you build a component, write the full Javascript/Logic required to make tabs switch, modals open, or sliders move.
3. **No External Assets Dependencies**: Do not use external image URLs that might break (like Unsplash links that could expire). Use beautiful SVG shapes, CSS gradients, or Lucide/FontAwesome icons rendered inline to build the visuals.

4. **Bold Semantics & Performance:** Extreme creativity is never an excuse for sloppy code. Structure layouts with semantic HTML5 elements (`<nav>`, `<main>`, `<article>`, `<aside>`) to maintain clean accessibility (WCAG). Optimize animations by strictly utilizing hardware-accelerated properties (`transform`, `opacity`) to ensure the interface maintains a buttery-smooth 60fps, no matter how complex the visual effects.

NEVER use generic AI-generated aesthetics like overused font families (Inter, Roboto, Arial, system fonts), cliched color schemes (particularly purple gradients on white backgrounds), predictable layouts and component patterns, and cookie-cutter design that lacks context-specific character[cite: 1].

Interpret creatively and make unexpected choices that feel genuinely designed for the context[cite: 1]. No design should be the same[cite: 1]. Vary between light and dark themes, different fonts, different aesthetics[cite: 1]. 

**IMPORTANT**: Match implementation complexity to the aesthetic vision[cite: 1]. Maximalist designs need elaborate code with extensive animations and effects[cite: 1]. Minimalist or refined designs need restraint, precision, and careful attention to spacing, typography, and subtle details[cite: 1]. Elegance comes from executing the vision well[cite: 1].
